## Quick Plugin Template

```python
"""
Project : Converigo
Author  : [Your Name]
Version : 3.0.0

Source -> Target Plugin
"""

from pathlib import Path
from app.engines.{category}_engine import {Category}Engine
from app.plugins.base import ConverterPlugin


class SourceToTargetPlugin(ConverterPlugin):

    slug = "source-to-target"
    name = "Source to Target"
    description = "Convert source format to target format."
    category = "{category}"
    engine = "{category}"
    icon = "🔄"

    popular = True
    featured = False

    source_formats = ["source"]
    target_formats = ["target"]

    goal = "goal_name"
    use_case = "Best when users need..."
    priority = 75
    quality = 85
    compatibility = 90
    estimated_saving = 30
    badge = "Badge Name"

    seo_title = "Source to Target Converter | Converigo"
    seo_description = "Convert source format to target format quickly and easily."

    async def convert(
        self,
        source_path: Path,
        target_format: str,
    ) -> Path:
        if not self.supports(source_path.suffix, target_format):
            raise RuntimeError("Plugin only supports Source -> Target.")

        engine = {Category}Engine()
        return await engine.convert(
            source_path=source_path,
            target_format=target_format,
        )
```

## Contract Validation Checklist

Every `.contract.json` file must have:

- [ ] `id` - Unique identifier (string, non-empty)
- [ ] `slug` - URL-friendly ID (string, non-empty, unique)
- [ ] `name` - Display name (string, non-empty)
- [ ] `category` - Category type (string, non-empty)
- [ ] `description` - Short description (string, non-empty)
- [ ] `input_formats` - Input formats (non-empty array)
- [ ] `output_formats` - Output formats (non-empty array)
- [ ] `accepted_mime_types` - MIME types (non-empty array)
- [ ] `max_upload_size` - Max size in bytes (positive integer)
- [ ] `conversion_engine` - Engine type (string, non-empty)
- [ ] `landing_path` - URL path (string, non-empty)
- [ ] `canonical_url` - Full canonical URL (string, non-empty)
- [ ] `seo_status` - SEO status (string: "ready")
- [ ] `schema_status` - Schema status (string: "ready")
- [ ] `faq_status` - FAQ status (string: "ready")
- [ ] `regression_sample` - Test file path (string, non-empty)
- [ ] `supported_platforms` - Platforms (non-empty array, e.g., ["web"])
- [ ] `lifecycle_status` - Status (string: "active", "beta", or "deprecated")

## HTTP Route Patterns

```
POST /upload
  - Upload a file
  - Returns: {"status": "success", "filename": str}

GET /tools/{slug}
  - Get landing page for converter
  - Returns: HTML page with landing context

POST /convert/{converter_id}
  - Perform conversion
  - Returns: File download

GET /
  - Home page with popular converters
  - Returns: HTML page

GET /plugins
  - List all available plugins
  - Returns: JSON with plugin info

GET /recommend
  - Get recommendations
  - Returns: JSON with recommended converters

GET /api/tools/{slug}
  - Get converter data as JSON
  - Returns: Tool data JSON
```

## URL Patterns

- Landing page: `https://converigo.com/{slug}`
- API endpoint: `/api/tools/{slug}`
- Upload: `/upload`
- Convert: `/convert/{slug}`

Example:
- `https://converigo.com/png-to-jpg`
- `https://converigo.com/pdf-to-word`
- `https://converigo.com/mp4-to-mp3`

## File Naming Convention

```
app/data/converters/
├── {slug}.contract.json          # Converter contract (required)
├── {slug}.json                   # Converter data (optional)
└── {slug}.metadata.json          # Metadata (optional, fallback only)

app/plugins/{category}/
└── {slug}.py                     # Plugin implementation

Example:
├── png-to-jpg.contract.json
├── png-to-jpg.json
├── pdf-to-word.contract.json
├── pdf-to-word.json
├── pdf-to-word.metadata.json
```

## Slug Naming Convention

Format: `{source-format}-to-{target-format}`

Rules:
- Lowercase only
- Hyphen-separated
- Must be unique
- Used for URLs, files, and IDs

Examples:
- `png-to-jpg`
- `pdf-to-word`
- `mp4-to-mp3`
- `excel-to-pdf`
- `webp-to-png`

## Key Insights

1. **Dual Source of Truth:**
   - Contracts define what MUST be present (machine-enforced)
   - Data files define what SHOULD be present (user-curated content)

2. **Smart Defaults:**
   - Missing data fields auto-generated from slug and contract
   - Ensures no broken pages, but encourages data enrichment

3. **Scoring System:**
   - Related converters scored by match type
   - Same input (4pts) > category (3pts) > cluster (2pts) > output (1pt)
   - Provides intelligent recommendations

4. **Quality Validation:**
   - Audit service checks all layers simultaneously
   - Contract → Landing → Knowledge → SEO → Sitemap
   - Produces actionable quality metrics

5. **Plugin Extensibility:**
   - New converters add: contract + data + plugin
   - Service layer auto-discovers and registers
   - No code changes needed in routing or core

6. **Content Organization:**
   - Contracts are version-controlled reference data
   - Data files can be edited without touching code
   - Knowledge is generated deterministically
   - All SEO generated systematically
