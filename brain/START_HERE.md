# Converigo Brain — Start Here

This folder is the Converigo Brain: the authoritative source for project state, checkpoint history, release decisions, and product guidance.

Use this collection to understand what has been built, why it exists, and what is ready for release.

## What this contains

- `PROJECT_STATE.md` — current product state, phase, milestone, and package scope.
- `CHECKPOINTS.md` — checkpoint definitions, status, and release gate history.
- `DECISIONS.md` — decision log for architecture, product scope, and release criteria.
- `docs/RELEASE_NOTES.md` — the official release note for the current checkpoint.

## How to use it

1. Read `PROJECT_STATE.md` first to understand the current mission and milestone.
2. Review `CHECKPOINTS.md` for the status of Checkpoint C1 and the release gate.
3. Consult `DECISIONS.md` before making changes to ensure consistency with project direction.
4. Update `docs/RELEASE_NOTES.md` only when a checkpoint is ready to ship.
