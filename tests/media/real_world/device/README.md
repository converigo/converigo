# Device Samples

## Purpose
Store media samples generated by specific devices such as smartphones, cameras, and screen recorders.

## Sample naming convention
Use the pattern: device_<device_name>_<container>_<codec>_<audio>.<ext>

## Required metadata
- device model or class
- container
- codec
- audio
- capture source
- expected validator result

## Priority
- P0: common mobile and camera files
- P1: legacy and specialist hardware samples
- P2: niche device-specific cases

## Future regression policy
Any device-specific false negative or false positive must become a regression case.
