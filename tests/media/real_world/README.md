# Real World Samples

## Purpose
Capture media that reflects real user and production workflows.

## Sample naming convention
Use the pattern: <source>_<container>_<codec>_<audio>.<ext>

## Required metadata
- source type
- device or editor
- container
- codec
- audio
- expected validator result
- expected conversion result

## Priority
- P0: baseline consumer and audio samples
- P1: editor and broadcast-style samples
- P2: niche or uncommon real-world cases

## Future regression policy
Promote any real-world failure into the regression corpus.
